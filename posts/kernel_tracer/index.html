My Simple Utility for Kernel Function Graph Tracing

Posted Jan 10, 2022
By Lior Ribak
2 min read

Have you ever wondered how come strace is such a popular tool, while all it shows you are only the syscalls a program is calling and nothing more?

Well, in most cases it‚Äôs enough. But sometimes, even rarely, a programmer will also want to inspect what happens under the kernel‚Äôs hood inside the syscalls.

I have been investigating kernel behavior on some basic tasks, like module loading for example, so my mission was to figure out the code path that is taken inside the kernel upon a simple invocation of the insmod command.

strace wasn‚Äôt very helpful: it only displayed a call for an init_module syscall, but I wanted the function graph inside it!

After some web searches, and I didn‚Äôt find any simple tool fitted to my problem.

The closest thing was trace-cmd, which uses the ftrace API for the job and creates a function graph as I wish. But I need to run it in a separate shell from my traced program, and also pass it my program‚Äôs PID, which is problematic for quickly exiting simple utilities like insmod.

As a result, and as any decent bored engineer would do, I created a wrapper utility to exactly fit my needs üòÅ

Welcome kernel_function_trace !

bad name, but ktrace was too presumptuous.

Shortly, it‚Äôs 50 lines of a C-code wrapper around trace-cmd, with some forking and signaling (and a tiny bit of sleeping), that makes kernel function tracing a little bit easier

It wraps around any program the same way strace does:
